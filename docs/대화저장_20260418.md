### 3. 소상공인365 API 탐색 (이번 세션 주요 이슈)

**발견된 사실:**
- 간단분석 리포트: 행정동 단위 + 업종 세분화(국/탕/찌개류 등) + 배달 데이터 포함
- 실제 동작하는 API 엔드포인트 확인:
  ```
  GET https://bigdata.sbiz.or.kr/gis/simpleAnls/getAvgAmtInfo.json
  파라미터: admiCd={행정동8자리코드}&upjongCd={소분류코드5-6자리}
  인증: Authorization: JWT토큰 (만료: ~1시간)
  ```
- Response 주요 필드: saleAmt(월매출), siAmt(서울평균), prevMonGuAmt(전월구평균), annualSales(12개월추이), topFive(전국상위5개동), storeCntAdmin(동별점포수추이)

**장벽:**
1. JWT 토큰 1시간마다 만료 → 대량 수집 중 재인증 필요
2. 업종 소분류 코드 목록 API 미발견 (getTpbizClsCode.json 경로 불일치)
3. 서버(Cowork) 환경에서 403 차단 → 브라우저 콘솔에서만 접근 가능
4. 서울 424동 × 50업종 = 21,200회 호출 필요 → 현실적으로 수작업 불가

**알려진 업종 코드 (확인됨):**
- I20102 = 국/탕/찌개류
- I20401 = ? (copy-as-fetch에서 확인, 이름 미상)

**판단:**
- 기술적 가능성 40~50%
- 이미 서울 열린데이터광장 CSV (424동×63업종) 보유 중 → 소상공인365 없이도 동 단위 업그레이드 가능
