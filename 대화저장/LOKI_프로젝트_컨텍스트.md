## 데이터 구조

### META (테마1 내 임베드)
```js
META = { "업종명": { p, s, e, c, o, cat } }
```
- p: 평수 (pyeong)
- s: 면적 스케일 파라미터
- e: 직원 수
- c: 원가율 (0~1)
- o: 기타비용율 (%)
- cat: 업종 카테고리 (소매/음식·숙박/서비스·기타)

### LOOKUP (테마1 내 임베드)
```js
LOOKUP = { "구": { "업종": { a, m, r, s, g, n } } }
```
- a: 상위 매출 (만원/월)
- m: 중위 매출 (만원/월)
- r: 임대료 (만원/㎡/월)
- s: 업소 수
- g: 성장률 (%)
- n: 비고

### T2 (테마2 내 임베드)
```js
T2 = { "업종": { "구": { sc, ps, pc, pg, pe, a, s, g } } }
```
- sc: 종합 적합도 점수 (0~100)
- a: 월평균매출 (만원, LOOKUP.m과 동기화)

## 수익 계산 로직
```js
const MIN_WAGE = 10030, HOURS = 209;
rent  = row.r × meta.p × 3.3 / 10          // 만원/월
labor = MIN_WAGE × HOURS × meta.e / 10000   // 만원/월
net   = midSales × (1 - c - o/100) - rent - labor
```
